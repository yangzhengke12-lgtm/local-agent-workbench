# Multi-Agent 系统 v4 迭代清单（10 个关键卡点）

> 2026-05-29 分析 · 基于 v3 现状

## 已具备的能力
- 多厂商路由 + 失败自动升级（DeepSeek → GPT）
- 任务看板 + 知识库 + 评分体系（JSON 持久化）
- 圆桌讨论（两轮并行 + 交叉回应）
- 副经理复核（独立判断 + 质疑权）
- AI 生成 Pipeline + 动态分工
- Worker 间通信（ask_coworker）
- 会话记忆持久化

---

## 10 个关键缺口

### 1. 任务分解是一次性的，不会动态调整
project_setup() 在项目启动时拆解一次，存为静态 JSON。Worker 执行中发现了新的子问题，无法注入 Pipeline。需要：执行中持续 replan。

### 2. 没有"验证→驳回→重试"闭环
evaluate_result 只评分，不触发重做。Sophia 说"差"，系统不会让 Alex 按反馈重写。需要：verify-reject-retry 自动循环。

### 3. 没有收敛模式
做不到"持续找 bug，直到连续 N 轮没有新发现"。当前是请求-响应模型，不是 loop-until-convergence。

### 4. Worker 输出是自由文本，没有结构化合约
Worker 返回文字，Manager 靠 prompt 解析。没有 JSON Schema 约束和程序化校验。需要：structured output contracts。

### 5. 复杂度判断是关键词匹配，不是语义理解
COMPLEXITY_SIGNALS 是正则列表。50 行重命名匹配"重构"触发 GPT-5.5，真正的复杂并发 bug 反而被低估。需要：LLM 自判断复杂度。

### 6. 没有 Pipeline 运行时引擎
depends_on 存在 JSON 里但不强制——A 失败了 B 照样启动。需要：DAG 依赖感知的调度器。

### 7. Worker 之间没有文件系统隔离
所有 Worker 共享文件系统，读写可能竞态。需要：git worktree 或临时目录隔离。

### 8. 没有 Token 预算感知
Worker 没有"最多 N token"的上限，可能无限循环消耗。需要：熔断和预算控制。

### 9. 没有对抗验证
副经理复核 Manager 决策，不是 Worker 产出。没有 agent 的角色是"证明你的结果是错的"。需要：adversarial verifier。

### 10. 没有增量 Replan
Pipeline 有 5 步，第 2 步发现计划错了，3-5 已分派、不会自动调整。需要：依赖链感知的增量重规划。

## 优先级建议

| 优先级 | 缺口 | 理由 |
|--------|------|------|
| P0 | #2 验证闭环 | 本质改变，让系统从"调度器"变成"自动驾驶" |
| P0 | #4 结构化输出 | 是整个闭环的基础，没有合约 verify 无法程序化 |
| P1 | #3 收敛模式 | 解锁"质量驱动"的执行策略 |
| P1 | #6 Pipeline 引擎 | 让 project_setup 不再是"写了不看"的文档 |
| P2 | 其余 | 锦上添花，按需迭代 |
